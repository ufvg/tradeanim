tradeanim

A Manim-like Python library for creating animated trading chart videos. Render publication-quality chart animations with clean, declarative code.

v0.2.0 Python 3.9+ matplotlib + ffmpeg

# Getting Started

Installation

Install tradeanim from PyPI or from source.

shell# Install from PyPI
pip install tradeanim

# Or install from source
git clone https://github.com/tradeanim/tradeanim.git
cd tradeanim && pip install -e .

# System dependency — ffmpeg is required
brew install ffmpeg    # macOS
sudo apt install ffmpeg  # Ubuntu / Debian

Quick Start

Create your first animated chart video in under 10 lines of code. Define a scene, load OHLC data, play animations, and render to an mp4. 

pythonfrom tradeanim import *

class MyScene(Scene):
    def construct(self):
        chart = self.load_chart("data.csv")
        self.play(CandlesAppear(chart), duration=3)
        self.wait(1)

MyScene(fps=30, width=1920, height=1080).render("output.mp4")
How it works The construct() method builds an animation timeline. When render() is called, tradeanim replays the timeline frame-by-frame, rendering each frame with matplotlib and piping raw RGB data to ffmpeg for encoding.

# Core Concepts

Scene

The Scene is the main entry point. Subclass it and override construct() to build your animation timeline. 

pythonclass Scene:
    def __init__(self, config: Optional[RenderConfig] = None, **kwargs)

Key Methods

Method Return Description
load_chart(csv_path, **kwargs) Chart Load OHLC data from CSV
play(*animations, duration=None, delay=0.0) — Schedule animations; advances timeline
play_with_previous(*animations, duration=None, offset=0.0) — Overlap with previous play() call; does not advance timeline
wait(duration=1.0) — Pause / advance timeline
add_element(element) — Manually add an element to the scene
remove_element(element) — Manually remove an element from the scene
render(output_path="output.mp4") — Build and render to video

Properties

Property Type Description
total_duration float Total timeline length in seconds
visible_elements list Elements where visible=True and opacity > 0
camera Camera The Camera instance
chart Chart | None The loaded Chart (or None)

Example showing play_with_previous for overlapping animations:

pythonclass OverlapDemo(Scene):
    def construct(self):
        chart = self.load_chart("data.csv")
        title = Title("My Chart")

        self.play(FadeIn(title), duration=0.5)
        self.play(CandlesAppear(chart), duration=4)
        # Fade out title while candles are still appearing
        self.play_with_previous(FadeOut(title), duration=0.8, offset=2.0)
        self.wait(1)

Animation Base Class

All animations extend the Animation base class. 

pythonclass Animation:
    def __init__(self, duration=1.0, easing=None)

Override Points

Method Description
on_start(scene) Called once when animation begins
on_update(scene, progress) Called every frame with eased progress (0.0 to 1.0)
on_finish(scene) Called once when animation completes

Default easing is ease_out_cubic for smooth deceleration.

Example custom animation:

pythonfrom tradeanim import Animation
from tradeanim.easing import ease_in_out_sine

class PulseOpacity(Animation):
    def __init__(self, element, min_val=0.3, max_val=1.0):
        super().__init__(duration=2.0, easing=ease_in_out_sine)
        self.element = element
        self.min_val = min_val
        self.max_val = max_val

    def on_update(self, scene, progress):
        import math
        t = math.sin(progress * math.pi * 2)
        self.element.opacity = self.min_val + (self.max_val - self.min_val) * (t * 0.5 + 0.5)

Timeline System

The timeline is built by three methods that control when animations start and how the current time advances:

  • play() — schedules animations at the current time and advances the timeline by their duration.
  • play_with_previous() — schedules at the previous play() start time + offset; does not advance the timeline.
  • wait() — advances the timeline without any animation.
  • delay parameter in play() — staggers multiple animations within the same call.
textTimeline:
0s          1s          2s          3s          4s
|-- play(A, duration=2) --|
            |-- play(B, duration=2) --|
                    |-- play_with_previous(C, offset=1) --|
                                      |-- wait(1) --|
Tip: Think of play() as “do this next” and play_with_previous() as “do this at the same time as the last thing.” The offset parameter lets you fine-tune exactly when the overlap begins.

Post-Processing Pipeline

Five post-processing effects applied after each frame. Requires Pillow.

  • Bloom / Glow — Gaussian blur blended with original for a dreamy glow effect. Controls: bloom_enabled, bloom_radius (blur size), bloom_intensity (blend strength).
  • Vignette — Radial darkening from edges toward center. Controls: vignette_enabled, vignette_strength.
  • Color Grading — Adjust brightness, contrast, and saturation. Controls: color_grading dict with keys "brightness", "contrast", "saturation" (all default 1.0).
  • Chromatic Aberration — RGB channel splitting for a lens distortion look. Red channel shifts left, blue shifts right. Controls: chromatic_aberration_enabled, chromatic_aberration_offset (pixel shift).
  • Lens Distortion — Barrel (negative k) or pincushion (positive k) distortion. Controls: lens_distortion_enabled, lens_distortion_k.

Full cinematic example:

pythonpp = PostProcessConfig(

    chromatic_aberration_enabled=True,
    chromatic_aberration_offset=2.0,
)
